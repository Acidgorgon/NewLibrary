Installation

Load the library from the CDN with a single loadstring call. The return value is your Library module.

CDN Loadstringlua
local Library = loadstring(game:HttpGet("https://gist.githubusercontent.com/Acidgorgon/2c0654b99a77b27e801ca6eb952a1e6c/raw/33f0a4f978d700fab58fd2b4374004cf7a87e560/UILib"))()
ℹ
The returned Library table exposes Library.new(), Library:InitLoad(), Library:Notify(), and Library:SetTheme() at the global level.

InitLoad

Displays a full-screen cinematic splash screen with a particle field, animated segment bar, glitch-typewriter title, and terminal-style status log. Call this before Library.new().

Library:InitLoad(options: table)
Key Type Description
Title string Large glitch-typewriter title text. Default: "NEXUS HUB"
SubTitle string Secondary line shown below title (version, author). Default: ""
Icon number|string Asset ID or rbxassetid URI for the logo. Displayed above the title. Optional.
Duration number Total splash duration in seconds (minimum 2). Default: 5
AccentColor Color3 Color used for segments, corners, particles. Default: Color3.fromRGB(0, 210, 150)
BackgroundColor Color3 Backdrop color. Default: Color3.fromRGB(5, 8, 13)
Statuses {string} Array of terminal log strings to cycle through. Each is typewritten as the bar progresses.
Silent boolean If true, skips the UI entirely and sets Library.IsSilent = true. Also auto-detected from saved config.
Examplelua
Library:InitLoad({
    Title       = "NEXUS HUB",
    SubTitle    = "v0.06 — by acidg",
    Icon        = 10734914443,
    Duration    = 5,
    AccentColor = Color3.fromRGB(0, 200, 255),
    Statuses    = {
        "Verifying environment...",
        "Loading modules...",
        "Authenticating...",
        "Building interface...",
        "Ready."
    }
})
⚠
Silent Execution: If the player has a saved config with SilentExecution = true, InitLoad will automatically set Library.IsSilent = true and return early — even if Silent is not passed. All subsequent UI calls are no-ops in silent mode.

Library.new

Creates and returns a Window — the root container for all tabs and widgets. Registers cleanup automatically via getgenv()._NextLevel_Garbage.

local Window = Library.new(Name, Size, ToggleKey, Mode, Icon)
Parameter Type Description
Name string Window title shown in the header. Also used as the GC namespace and config folder prefix.
Size UDim2 Initial window dimensions. Minimum enforced at 400×300 by the resize grip. Default: UDim2.fromOffset(520, 560)
ToggleKey Enum.KeyCode | Enum.UserInputType Key/input to show or hide the window. Default: Enum.KeyCode.RightControl
Mode string "Sidebar" — icon sidebar with two-column sections. "Classic" — top tab bar with single column. Default: "Classic"
Icon number|string Asset ID for the logo shown next to the title. Optional.
Sidebar Modelua
local Window = Library.new(
    "My Hub",
    UDim2.new(0, 560, 0, 580),
    Enum.KeyCode.RightControl,
    "Sidebar",
    10734914443
)
Classic Modelua
local Window = Library.new(
    "My Hub",
    UDim2.new(0, 520, 0, 480),
    Enum.KeyCode.Insert,
    "Classic"
)
ℹ
Window properties accessible after creation: Window.Name, Window.Size, Window.KeyBind, Window.Flags, Window.Setters, Window.ConfigFolder, Window.Presets, Window.NotificationHistory, Window.IsSilent.

Window API

All methods available on the Window object returned by Library.new().

Notify window

Window:Notify(title: string, text: string, duration?: number)

Displays an animated slide-in notification in the top-right corner. Entries are recorded to Window.NotificationHistory. No-op in silent mode.

Examplelua
Window:Notify("Success", "Operation completed.", 3)
 Window:Notify("Warning", "Rate limit approaching.", 5)
 Window:Notify("Error",   "Connection timed out.",   8)

Dialog window

Window:Dialog(title: string, message: string, buttons: {string}, callback: (choice: string) -> ())

Renders a blocking modal with a backdrop overlay. The callback receives the text of the button clicked. Animates in with a spring scale and fades out on close.

Examplelua
Window:Dialog("Confirm Reset", "This will delete all saved data. Are you sure?",
    {"Yes, Reset", "Cancel"},
    function(choice)
        if choice == "Yes, Reset" then
            -- wipe data
        end
    end
)

Unload window

Window:Unload()

Cleanly destroys all GUI instances, disconnects all RBXScriptConnections, cancels all running threads, and calls all registered :OnUnload() callbacks. Uses the global GC table keyed by Window.Name.

Examplelua
Window:Unload()

OnUnload window

Window:OnUnload(callback: () -> ())

Registers a cleanup function to run when :Unload() is called. Use this to stop game loops or restore modified properties.

Examplelua
local flyLoop = task.spawn(function()
    while true do
        -- fly logic
        task.wait()
    end
end)

Window:OnUnload(function()
    task.cancel(flyLoop)
    -- restore player speed, etc.
end)

Connect window

Window:Connect(signal: RBXScriptSignal, callback: function) → RBXScriptConnection

Connects a signal and registers the connection in the GC table. Connections are automatically disconnected on :Unload(). Prefer this over bare :Connect() inside your script.

Examplelua
Window:Connect(game.Players.LocalPlayer.CharacterAdded, function(char)
    Window:Notify("Respawned", "Character loaded.", 2)
end)

SetPanicKey window

Window:SetPanicKey(key: Enum.KeyCode)

Registers an emergency key that calls :Unload() instantly, removing all traces of the GUI.

Examplelua
Window:SetPanicKey(Enum.KeyCode.End)  -- DELETE key unloads everything

AddTooltip window

Window:AddTooltip(element: GuiObject, tipText: string)

Attaches a hover tooltip to any GuiObject created by the library. Follows the mouse cursor and auto-hides on leave.

Examplelua
local myBtn = Section:Button("Teleport", function() end)
Window:AddTooltip(myBtn, "Teleports you to spawn point.")

GetFlag / IgnoreFlag window

Window:GetFlag(flag: string) → any
Window:IgnoreFlag(flag: string)

GetFlag reads the current value for a named flag. IgnoreFlag marks a flag so it is excluded from config saves/loads.

Examplelua
-- Read a flag at any point:
local fov = Window:GetFlag("FovFlag")  -- returns current slider value

-- Prevent a flag from being persisted:
Window:IgnoreFlag("SensitiveFlag")

ServerHop window

Window:ServerHop(placeId: number, jobId?: string, scriptToQueue?: string)

Saves all current flags to teleport_state.json and sets autoload to that file, then teleports the player. If scriptToQueue is provided, it is queued via queue_on_teleport.

Examplelua
-- Hop to a random server in the same game:
Window:ServerHop(game.PlaceId)

-- Hop to a specific server with a re-inject script:
Window:ServerHop(game.PlaceId, someJobId, [[
    loadstring(game:HttpGet("https://my.cdn/script.lua"))()
]])

SetTheme / ApplyPreset window

Window:SetTheme(overrides: table)
Window:ApplyPreset(name: string)

SetTheme merges a partial theme override and fires the reactive ThemeEvent, tweening all bound elements. ApplyPreset applies one of the six built-in color presets.

Examplelua
Window:SetTheme({
    AccentColor = Color3.fromRGB(120, 80, 255),
    LineColor   = Color3.fromRGB(150, 100, 255),
})

Window:ApplyPreset("Cyberpunk") -- or "Midnight", "Toxic", "Ocean", "Classic", "Ghost"

RGB Chroma Loop

The Settings tab provides a built-in "RGB Chroma Accents" toggle that continuously cycles LineColor through hue using Color3.fromHSV. You can replicate this manually:

Manual RGB Looplua
task.spawn(function()
    while true do
        local hue = tick() % 10 / 10
        Window:SetTheme({ LineColor = Color3.fromHSV(hue, 1, 1) })
        task.wait(0.1)
    end
end)

Flags & Setters

Every widget with a flag parameter registers itself in two tables on the Window. Understanding this system is key to programmatic control and config persistence.

Window.Flags — a flat key-value table mapping every flag name to its current value. Readable at any time via Window.Flags["MyFlag"] or Window:GetFlag("MyFlag").
Window.Setters — maps every flag name to a setter function. Calling Window.Setters["MyFlag"](newValue) is equivalent to the user interacting with the widget — it updates the visual, the flag, and fires the callback.
Reading & Writing Flagslua 
-- Create a toggle with a flag:
Section:Toggle("Aimbot", false, "AimbotFlag", function(state)
    print("Aimbot:", state)
end)

-- Read it anywhere:
local isEnabled = Window:GetFlag("AimbotFlag")          -- false
local isEnabled = Window.Flags["AimbotFlag"]            -- same thing

-- Set it programmatically (fires callback + updates visual):
Window.Setters["AimbotFlag"](true)

-- Prevent from being saved in configs:
Window:IgnoreFlag("AimbotFlag")
✦
Config saving iterates over Window.Flags and serialises every key not in Window.IgnoredFlags. Loading calls each matching Window.Setters[flag] in turn. This is how all widget states are persisted automatically.

Tabs

Tabs are primary navigation categories. In Sidebar mode, tabs display as icon-label rows in the left sidebar. In Classic mode, they display as a horizontal tab bar across the top.

local Tab = Window:Tab(name: string, iconId?: number|string)
Parameter Type Description
name string Tab label text displayed in the navigation.
iconId number|string Roblox asset ID for the tab icon. Highly recommended in Sidebar mode for best aesthetics.
Examplelua
local CombatTab  = Window:Tab("Combat",   10734914443)
local VisualTab  = Window:Tab("Visuals",  9886724082)
local MiscTab    = Window:Tab("Misc",     11963373104)
local SettingsTab = Window:Tab("Settings", 3926307971)
◦
The first tab created is automatically shown and highlighted when the window opens.

Sections

Sections are bordered containers that hold widgets. In Sidebar mode, sections alternate between the left and right columns automatically for a two-column grid layout.

local Section = Tab:Section(title: string)

Creates a bordered section with a floating title label. Section height is automatically driven by its content via UIListLayout.AbsoluteContentSize.

Examplelua
local AimSection   = CombatTab:Section("Aimbot")
local ESPSection   = VisualTab:Section("ESP Settings")
local MiscSection  = MiscTab:Section("Miscellaneous")

-- All widgets go into a Section:
AimSection:Toggle("Enable Aimbot", false, "AimFlag", function(state) end)

Layout Elements

Structural helpers for organising content within sections.

Separator

Section:Separator(label?: string)

A thin 1px horizontal rule. With a label, shows flanking lines around centred text. Without, a simple full-width line.

Examplelua
Section:Separator()                  -- bare line
Section:Separator("ADVANCED")

Spacer

Section:Spacer(height?: number)

Invisible vertical padding in pixels. Default: 10.

Examplelua
Section:Spacer(20)  -- 20px vertical gap

Label → controller

Section:Label(text: string) → LabelController

Short single-line text in TextColor. Supports TextWrapped. Returns an object with :SetText(str) for dynamic updates.

Examplelua
local lbl = Section:Label("Status: Idle")

-- Update dynamically:
lbl:SetText("Status: Active")

-- Or directly set Roblox property:
lbl.TextColor3 = Color3.new(0, 1, 0)

Paragraph → controller

Section:Paragraph(text: string) → ParagraphController

Multi-line wrapped text in SubTextColor. Auto-sizes height. Returns :SetText(str).

Examplelua
local info = Section:Paragraph("This feature is in beta. Report any bugs in the Discord server.")
info:SetText("Updated: " .. os.date("%d/%m/%Y"))

Alert

Section:Alert(text: string, alertType?: "info"|"warn"|"error"|"success")

A coloured inline notification box with a left accent bar and an icon glyph. Height scales to fit wrapped content.

Examplelua
Section:Alert("New update available — v0.07", "info")
Section:Alert("High ping detected. Performance may be reduced.", "warn")
Section:Alert("Authentication failed. Re-inject to fix.", "error")
Section:Alert("All systems operational.", "success")

Buttons

Clickable action elements with hover animations and click sound feedback.

Button → TextButton

Section:Button(text: string, callback?: () -> ()) → TextButton

Full-width button. Returns the raw TextButton instance so you can pass it to :AddTooltip() or modify it directly.

Examplelua
local resetBtn = Section:Button("Reset Character", function()
    game.Players.LocalPlayer.Character:BreakJoints()
end)

Window:AddTooltip(resetBtn, "Instantly kills your character.")

CreateButtonRow

Section:CreateButtonRow(text1, callback1, text2, callback2) → btn1, btn2

Places two equal-width buttons side by side in a single row. Returns both TextButton instances.

Examplelua
local accept, decline = Section:CreateButtonRow(
    "Accept", function()
        Window:Notify("Accepted", "Terms accepted.", 3)
    end,
    "Decline", function()
        Window:Notify("Declined", "Terms declined.", 3)
    end
)

Toggles & Switches

Boolean state widgets. All register to Window.Flags and Window.Setters.

Toggle

Section:Toggle(text, default?: boolean, flag?: string, callback?: (boolean)->void)

Checkbox-style toggle with a filled square. Also updates the Keybind HUD when active.

Examplelua
Section:Toggle("Aimbot", false, "AimbotState", function(state)
    aimbot.enabled = state
end)

-- Programmatically enable:
Window.Setters["AimbotState"](true)

ToggleBind

Section:ToggleBind(text, state?:bool, key?:KeyCode, stateFlag?:string, keyFlag?:string, callback?:(bool,string)->void)

Combines a toggle checkbox with a rebindable key button. Pressing the bound key toggles state in real-time. Both the state and key are independently persisted as separate flags. Callback receives (enabled, keyName).

Examplelua
Section:ToggleBind(
    "Fly",
    false,
    Enum.KeyCode.F,
    "FlyState",
    "FlyKey",
    function(enabled, keyName)
        if enabled then
            startFly()
        else
            stopFly()
        end
    end
)

Switch

Section:Switch(text, default?:bool, flag?:string, callback?:(bool)->void)

Animated sliding pill switch. Functionally identical to Toggle but uses a track-and-knob visual. Ideal for on/off settings where the sliding motion reinforces the "mode" metaphor.

Examplelua
Section:Switch("Silent Mode", false, "SilentF", function(state)
    game.SoundService.RespectFilteringEnabled = not state
end)

Sliders, Steppers & Bind

Numeric control widgets for continuous values, stepped values, and hotkey assignment.

Slider

Section:Slider(text, min, max, default?, decimals?, flag?, callback?)

A draggable fill bar for continuous numeric values. decimals controls float precision (0 = integers). Dragging and releasing fires the callback. The current value is displayed top-right.

Examplelua
-- Integer slider (0 decimals):
Section:Slider("FOV", 10, 180, 70, 0, "FovF", function(val)
    Camera.FieldOfView = val
end)

-- Float slider with 2 decimal places:
Section:Slider("Speed Multiplier", 0.1, 5.0, 1.0, 2, "SpeedF", function(val)
    humanoid.WalkSpeed = 16 * val
end)

Stepper

Section:Stepper(text, min, max, default?, step?, flag?, callback?)

Discrete − / + increment control. Best for values with fixed steps like gravity (multiples of 5) or hit delay (multiples of 50ms).

Examplelua
Section:Stepper("Gravity", 0, 196, 196, 5, "GravF", function(val)
    workspace.Gravity = val
end)

Section:Stepper("Hit Delay (ms)", 0, 1000, 100, 50, "HitDelayF", function(val)
    hitbox.delay = val / 1000
end)

Bind

Section:Bind(text, defaultKey?:KeyCode, flag?, callback?)

Standalone keybind listener. Click the button, then press any key to bind. Backspace or Escape clears the bind. The callback fires each time the key is pressed (not held).

Examplelua
Section:Bind("Teleport to Waypoint", Enum.KeyCode.T, "TpWaypointKey", function()
    teleportToWaypoint()
end)

Section:Bind("Panic Key", Enum.KeyCode.End, "PanicKey", function()
    Window:Unload()
end)

Text Inputs

Text entry widgets from single-line inputs to tag chip collections.

Input

Section:Input(text, placeholder?, flag?, callback?)

Single-line text box. Callback fires on FocusLost, not on every keystroke.

Examplelua
Section:Input("Target Username", "Enter player name...", "TargetNameF", function(text)
    targetPlayer = findPlayer(text)
end)

MultilineInput

Section:MultilineInput(text, placeholder?, lineCount?:number, flag?, callback?)

Multi-line text area. Height is derived from lineCount × 18 + 10 pixels. Default: 3 lines. Used internally by BuildExecutorTab.

Examplelua
Section:MultilineInput(
    "Custom Script",
    "print('Hello World')",
    8,            -- 8 visible lines
    "ScriptSource",
    function(code)
        loadstring(code)()
    end
)

TagInput → TagController

Section:TagInput(text, initialTags?:{string}, maxTags?:number, flag?, callback?)

Chips-style multi-value input. Press Enter or click + Add to add a chip. Click × on a chip to remove it. Duplicate entries and the max cap are automatically enforced. Section height grows to fit chips.

Parameter Type Description
text string Section label above the input.
initialTags {string} Pre-populated chip values. Default: {}
maxTags number Maximum number of chips. Default: 10
flag string Persists the tags array as a JSON array.
callback (tags: {string})->void Fired on add or remove with the full current array.
Examplelua
local tagCtrl = Section:TagInput(
    "ESP Whitelist",
    {"Friend1", "Ally2"},  -- initial chips
    20,                    -- max 20 players
    "EspWhitelistF",
    function(tags)
        espWhitelist = tags
    end
)

-- Controller API:
tagCtrl:GetTags()            -- returns copy of current tags
tagCtrl:SetTags({"A","B"})   -- replace all chips
tagCtrl:AddTag("NewPlayer")  -- add a single chip programmatically

Selection Suites

All dropdown variants return a DropdownController with a :Refresh() method for dynamic list updates.

Dropdown → DropdownController

Section:Dropdown(text, options:{string}, multi?:bool, default?, flag?, callback?)

Standard collapsible list. In multi = true mode, multiple options can be selected simultaneously — the flag value becomes an array. Dropdown opens with elevated ZIndex to render above other elements.

Single Selectionlua
local drop = Section:Dropdown(
    "Aim Part",
    {"Head", "Torso", "Left Arm", "Right Arm"},
    false,           -- single select
    "Head",          -- default
    "AimPartF",
    function(selected)
        aimPart = selected
    end
)

-- Refresh list later:
drop:Refresh({"Head", "Torso", "HumanoidRootPart"})
Multi Selectionlua
Section:Dropdown(
    "ESP Teams",
    {"Red", "Blue", "Green", "Yellow"},
    true,           -- multi select
    {"Red", "Blue"}, -- default selected
    "EspTeamsF",
    function(selected)
        -- selected is a {string} array
        espTeams = selected
    end
)

SearchableDropdown → DropdownController

Section:SearchableDropdown(text, options:{string}, default?, flag?, callback?)

Like Dropdown but includes an integrated search box that filters options in real-time. Essential for large lists (50+ items). Used internally by PlayerDropdown.

Examplelua
local gameNames = {"Arsenal", "Jailbreak", "Phantom Forces", "Da Hood", -- ...hundreds more
}

local gameDrop = Section:SearchableDropdown(
    "Select Game",
    gameNames,
    "",
    "ActiveGameF",
    function(selected)
        loadGamePreset(selected)
    end
)

-- Add entries dynamically:
table.insert(gameNames, "New Game")
gameDrop:Refresh(gameNames)

PlayerDropdown → DropdownController

Section:PlayerDropdown(text, includeSelf?:bool, flag?, callback?)

A SearchableDropdown pre-populated with all current server players. Automatically updates on PlayerAdded and PlayerRemoving. Connections are managed via Window:Connect() and cleaned up on unload.

Examplelua
Section:PlayerDropdown(
    "Target Player",
    false,           -- exclude self
    "TargetPlayerF",
    function(playerName)
        local target = game.Players:FindFirstChild(playerName)
        -- do something with target
    end
)

RadioGroup

Section:RadioGroup(label, options:{string}, default?, flag?, callback?)

Displays all choices vertically with radio dot indicators. Only one can be active at a time. Ideal when all options should be visible simultaneously (3–6 choices).

Examplelua
Section:RadioGroup(
    "Aimbot Mode",
    {"Silent Aim", "Lock-On", "Prediction", "Manual"},
    "Silent Aim",

Color & Images

ColorPicker

Section:ColorPicker(text, default?:Color3, alpha?:number, flag?, callback?)

Full HSV color picker with a saturation/value grid, a hue bar, and an alpha (transparency) bar. Expands/collapses inline. The color preview swatch in the header reflects alpha. Flag stores a table {R, G, B, A}.

Parameter Type Description
text string Label shown in the collapsed header.
default Color3 Initial color. Default: white.
alpha number Initial alpha 0–1. Default: 1 (opaque).
flag string Persists as {R, G, B, A} table. Setter accepts Color3 or the same table.
callback (color: Color3, alpha: number)->void Fires on any drag event with the new color and alpha.
Examplelua
Section:ColorPicker(
    "ESP Colour",
    Color3.fromRGB(255, 50, 50),
    0.85,          -- slightly transparent
    "EspColorF",
    function(color, alpha)
        espColor       = color
        espTransparency = 1 - alpha
    end
)

-- To set programmatically:
Window.Setters["EspColorF"]({R=0, G=1, B=0, A=1})

Image

Section:Image(assetId:number|string, height?:number) → ImageLabel

Embeds a Roblox image asset inline with ScaleType.Fit. Default height: 100 px. Pass either a numeric ID or a full rbxassetid:// URI.

Examplelua
Section:Image(10734914443, 120)      -- numeric ID, 120px tall
Section:Image("rbxassetid://123456789", 80)

Data Visualization

Widgets for displaying live or static data: charts, tables, bars, and status indicators.

Chart

Section:Chart(text, data:{{label,value}}, height?:number)

Animated vertical bar chart. Bars grow from 0 on first render using TweenService. Each bar displays its value above. Column labels appear below the chart area. Default height: 90 px.

Examplelua
Section:Chart("XP Per Hour", {
    {label = "Mon", value = 1200},
    {label = "Tue", value = 3400},

    {label = "Thu", value = 4100},

KeyValueTable → TableController

Section:KeyValueTable(rows:{{key, value}}) → KeyValueTableController

A compact two-column information grid. Odd/even rows alternate between MainColor and FrameColor for readability. Returns a controller to update individual values without rebuilding the table.

Examplelua
local infoTable = Section:KeyValueTable({
    {"Ping",    "0 ms"},
    {"FPS",     "0"},
    {"Players", tostring(#game.Players:GetPlayers())},
    {"Version", "v0.06"},
})

-- Update live:
game:GetService("RunService").RenderStepped:Connect(function(dt)
    infoTable:SetValue("FPS",  tostring(math.round(1/dt)))
    infoTable:SetValue("Ping", tostring(math.round(
        game.Players.LocalPlayer:GetNetworkPing() * 1000)) .. " ms")
end)

ProgressBar → ProgressBarController

Section:ProgressBar(text, min?, max?, default?, flag?) → ProgressBarController

Animated horizontal fill bar. The fill tweens smoothly on each :Set() call. Displays current / max in the top-right corner.

Examplelua
local xpBar = Section:ProgressBar("Experience", 0, 1000, 0, "XpF")

-- Update when XP changes:
function onXpChanged(newXp)
    xpBar:Set(newXp)
end

StatusIndicator → StatusController

Section:StatusIndicator(label, initialText?, initialColor?:Color3) → StatusIndicatorController

A labelled row with a coloured dot and status text on the right. The dot colour tweens on update. Perfect for connection status, API health, or feature active-state displays.

Examplelua

local apiStatus = Section:StatusIndicator("API Key", "Unchecked")

Utility Widgets

Higher-level interactive tools.

Console → ConsoleController

Section:Console(height?:number) → ConsoleController

A scrollable output log. Each line is prefixed with a [HH:MM:SS] timestamp. If already scrolled to the bottom, the view auto-scrolls on new entries. Default height: 120 px.

Examplelua
local console = Section:Console(160)

console:Print("Script initialised", Color3.fromRGB(100, 255, 100))
console:Print("Connecting to server...")
console:Print("ERROR: Connection refused", Color3.fromRGB(255, 80, 80))
console:Clear()  -- wipe all lines

Timer → TimerController

Section:Timer(text, seconds?, autoStart?:bool, onComplete?) → TimerController

Countdown widget with ▶ Start, ‖ Pause, and ↺ Reset buttons. The fill bar colour transitions yellow (<50%) → red (<25%) as time runs out. The onComplete callback fires when the timer reaches 0.

Parameter Type Description
text string Label shown top-left.
seconds number Total countdown duration. Default: 60
autoStart boolean Begin counting immediately on creation. Default: false
onComplete ()->void Called when remaining reaches 0.
Examplelua
local breakTimer = Section:Timer("Break Reminder", 300, false, function()
    Window:Notify("Break Time", "Take a 5 minute break!", 10)
end)

-- Controller API:
breakTimer:Start()      -- begin ticking
breakTimer:Pause()      -- pause
breakTimer:Reset()      -- reset to 300s
breakTimer:Set(120)     -- change duration to 2 minutes

Webhook

Section:Webhook(text, flag?, callback?)

Discord webhook URL input with an integrated Test button. Clicking Test POSTs a confirmation message to the URL using request(). The URL is persisted via the flag on every edit.

Examplelua
Section:Webhook("Kill Logger", "KillLoggerWebhookF", function(url)
    killWebhookUrl = url
end)

Grid

Section:Grid(text, items, columns?:number, callback?)

Arranges buttons in an evenly-spaced grid layout. Each item can be a plain string or a table {text = "...", id = "..."}. The callback receives the item's id (or the string itself if no id). Default columns: 3.

Section:Grid("Teleport Locations", {
    "Spawn", "Castle", "Market", "Forest", "Cave", "Rooftop"

    teleportTo(location)
end)

Section:Grid("Skins", {

Controller Reference

Return objects from widgets that expose programmatic control. All controllers are independent tables with methods — they do not inherit from any base class.

DropdownController — returned by Dropdown, SearchableDropdown, PlayerDropdown
:Refresh( newList: {string} ) — Rebuilds the option list and clears any search filter.
ProgressBarController — returned by ProgressBar
:Set( value: number ) — Animates the bar to the clamped value and updates the label.
StatusIndicatorController — returned by StatusIndicator
:Set( statusText: string, color?: Color3 ) — Tweens the dot colour and updates the right-side label.
KeyValueTableController — returned by KeyValueTable
:SetValue( key: string, value: any ) — Updates the value cell for the given key string.
ConsoleController — returned by Console
:Print( msg: string, color?: Color3 ) — Appends a timestamped line. Auto-scrolls if already at bottom.
:Clear() — Destroys all log lines and resets the line counter.
TimerController — returned by Timer
:Start()— Begin countdown (no-op if already running or at 0).
:Pause()— Halt the current countdown thread.
:Reset()— Restore remaining to original seconds and stop.
:Set( seconds: number )— Change the total duration and reset remaining.
TagInputController — returned by TagInput
:GetTags()— Returns a copy of the current tags array.
:SetTags( list: {string} )— Replaces all chips and rebuilds the display.
:AddTag( str: string )— Programmatically inserts a tag (honours max cap and dedup).
LabelController / ParagraphController — returned by Label and Paragraph
:SetText( text: string )— Updates the displayed text. Also proxies all raw TextLabel properties.

Config & Persistence

All config files are saved to {WindowName}_Configs/{PlaceId}/ as JSON. The Settings pre-built tab exposes all of this through a UI automatically.

SaveConfig

Window:SaveConfig(name: string)

Serialises all flags (excluding ignored flags) and the current window position to {name}.json.

LoadConfig

Window:LoadConfig(name: string)

Reads {name}.json and calls each matching Window.Setters[flag] in turn, updating all widget visuals and flags. Also restores the window position.

ListConfigs

Window:ListConfigs() → {string}

Returns an array of config names (without extension) in the config folder. Does not include teleport_state.

Manual Config Usagelua
-- Save current state:
Window:SaveConfig("default")

-- Load on next inject:
local configs = Window:ListConfigs()  -- {"default", "pvp", "safe"}
Window:LoadConfig("pvp")

-- Autoload: write the config name to autoload.txt
-- (BuildSettingsTab does this automatically via "Set Autoload" button)
✦
Autoload: If autoload.txt exists in the config folder and points to a valid config, BuildSettingsTab automatically loads it 0.5 seconds after the window is built. This is also checked by InitLoad to detect silent execution mode.

Pre-built Tabs

One-line commands that generate fully functional management tabs. Call after creating your Window.

BuildSettingsTab

Window:BuildSettingsTab(iconId?)

Generates a Settings tab with: config save/load/delete/export/import UI, autoload assignment, watermark toggle, keybind HUD toggle, RGB chroma toggle, background blur toggle, silent execution toggle, Unload button, and a status indicator. Runs autoload on build.

BuildThemeTab

Window:BuildThemeTab(iconId?)

Generates a Theme tab with a preset button row (Cyberpunk, Midnight, Toxic, Ocean, Classic, Ghost) and individual ColorPicker widgets for every Color3 key in THEMES. Includes a Reset to Default button.

BuildUtilityTab

Window:BuildUtilityTab(iconId?)

Generates a Utility tab with: Auto Clicker (toggle+bind+CPS slider), Rejoin / Server Hop buttons, and Copy Job ID.

BuildExecutorTab

Window:BuildExecutorTab(iconId?)

Generates an Executor tab with a 10-line multiline input and Execute / Clear buttons. Uses loadstring + task.spawn internally.

BuildNotificationLogTab

Window:BuildNotificationLogTab(iconId?)

Generates a Notifications tab showing the full Window.NotificationHistory with timestamps. Includes Refresh and Clear History buttons. History cap: 100 entries.

BuildChangelogTab

Window:BuildChangelogTab(data:table, iconId?)

Generates a Changelog tab. Pass a table where keys are version strings and values are note strings. Optionally include an Icon key in the table instead of passing iconId separately.
